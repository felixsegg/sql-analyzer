Generation Help

This window allows you to automatically generate SQL queries for a combination of selected LLMs and prompts.
  • Settings (âš™):
    Click the settings button to configure generation parameters:
    - Thread pool size: Number of LLM calls processed in parallel.
      • If you set the pool size too high, you may quickly run into API rate limits and failed requests.
      • If you set it too low, the generation process will take longer.
    - Repetitions: Number of times each prompt/LLM combination is generated.
      • Multiple repetitions help avoid cached responses by varying the temperature for each call.
    - LLM selection: Choose which large language models should be used for generation.
    - Prompt selection: Choose which prompts should be used for generation.
  • Start:
    Starts the batch generation process. Progress bars show the progress for each LLM.
  • Cancel:
    Stops the ongoing batch process.
  • Save:
    Saves all generated queries to the database. You can view them later in the "Generated Queries" overview.
The generation process uses the specified prompts, LLMs, and temperature settings to generate multiple SQL queries in parallel. Progress is displayed in real time.
